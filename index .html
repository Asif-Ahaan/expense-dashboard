Recommended Solutions

Prioritized action plan across immediate controls, process re-engineering, and technology improvements.

P1 Immediate P2 Short-term (30–90 days) P3 Medium-term (90–180 days)
P1

Restore Approval Workflow Immediately

Impact: ₹3.91T currently unapproved

Audit why 0 transactions have been approved in 3 years. Re-activate approval routing, assign approvers per cost center, and set SLA of 5 business days. Implement automated escalation if approver is unresponsive beyond 3 days.

Target: 100% approval rate within 30 days of submission
KPI: Avg. approval time ≤5 business days
Owner: CFO + Finance Ops
P1

Mandate Receipt Attachment — Hard Block

Impact: 5,722 transactions currently unverifiable

Change expense system to hard-block submission without receipt for any transaction above ₹500. Implement OCR auto-extraction from uploaded receipts to reduce manual effort. For small expenses (≤₹500), allow self-certification with manager acknowledgement.

Target: Receipt attachment rate >98%
KPI: Monthly % of no-receipt transactions
Tool: Expensify, SAP Concur, or native OCR integration
P1

Freeze & Recover Personal Expense Reimbursements

Recovery opportunity: ₹28.9 Billion

Immediately freeze the 1,380 rejected personal expense transactions. Cross-reference against payroll and initiate salary recovery or write-off process per HR policy. Strengthen detection by adding merchant category code (MCC) blocking at the card issuer level — not just keyword matching.

Target: 100% recovery or write-off resolution within 60 days
KPI: Personal expense rate <0.5% of transactions
Control: MCC-level card restrictions for personal categories
P2

Enforce Cost Center at Point of Entry

Impact: 4,807 transactions unmapped (32.9%)

Make cost center a required field with dropdown validation (no free-text) in the expense submission form. Pre-populate based on employee's department mapping in the HR system. Standardise all 23 department name variants into 6 canonical values via a lookup table in the master data.

Target: Cost center fill rate = 100%
KPI: Zero unclassified transactions per month
Effort: 1-time master data cleanse + form validation update
P2

Mandate Submitter Identity

Impact: 1,298 anonymous transactions

System should auto-populate submitted_by from the authenticated user session — no manual entry and no blank value permitted. For legacy anonymous records, initiate review to identify via card statement cross-referencing.

Target: 0 anonymous submissions
KPI: % transactions with submitter ID
Control: SSO-linked expense portal
P2

Resolve "UNKNOWN" Vendor Problem

Impact: ₹9.9B in 369 unidentified payee transactions

Require vendor selection from a pre-approved vendor registry. Disable free-text vendor entry. For onboarding new vendors, enforce a 5-day KYC process. The 369 existing UNKNOWN transactions require manual investigation and vendor retroactive identification.

Target: 0 UNKNOWN vendor transactions
KPI: Vendor registry coverage rate
Risk: Potential fraud in UNKNOWN pool — escalate to Internal Audit
P3

Implement FX Hedging Visibility

Impact: ₹387B in foreign currency exposure

Integrate expense system with treasury FX module to flag large foreign-currency transactions for hedging review. Establish monthly FX exposure reporting by currency and department. Set threshold alerts (e.g., USD transactions >$50K) for treasurer review.

Target: FX report available by 5th of each month
KPI: % of FX exposure hedged vs. unhedged
Savings: 1% FX improvement = ~₹2.2B annual saving
P3

Anomaly Detection & Continuous Monitoring

Prevents future leakage across all issue categories

Deploy rules-based + ML anomaly detection on the expense data: flag transactions with vendor-description mismatch, duplicate amounts within 7 days, submissions outside business hours, and expense spikes >3σ from peer median. Review weekly with Finance Ops.

Target: <1% false negative rate on fraud detection
KPI: Mean time to detect (MTTD) anomalies ≤24 hrs
Tool: Tableau / Power BI + Python rules engine
